Pixsplat

The game development introduced me to the programming world, I had to learn how to code to at least understand how games were created. However, it seems that I did not choose the simplest development area to take my first steps in this world, I quickly realized that it was not only necessary developer skills but also needed to be an artist.

My own Sokoban - One of my first tries of making games (2008)

In most of my projects when drawing and animating I ended up iterating with multiple drawing and animation programs looking for something I would never find, a program that can do the work for myself while I do what I can do right, coding.

But the search for the perfect software was not in vain and I noticed three interesting things:

  1. The most known options seemed too complex and expensive for me to draw low-resolution images.
  2. Most of the pixel art-oriented programs were lack of features requiring in most cases other programs to do some final tweaks to the job.
  3. The options that did not have any of the previous problems were not cross-platform or were web-based with not a very good performance.

What if I develop a drawing and animation program?

Pixsplat started as a program with the features that I would use for my games, but the project has evolved. The main goal is to cover the basic needs of an artist to work with pixel art, but it is also intended to provide some capabilities of more advanced image editors.

Pixsplat is written in C++ with wxWidgets, OpenGL and FreeImage for the UI, rendering the document canvas, and handling images respectively.
